Researcher data export

Enter the admin export key to download accumulated participant data from backend storage. The key is never stored anywhere by this page — you will need to re-enter it for each download, and the field is cleared immediately after every request.

Test records

Records submitted with test mode active (?test=1, gated by the server's ENABLE_TEST_MODE). Never mixed with production records.

Production records

Real participant submissions only.

CSV is one row per participant with analysis-friendly columns only (flattened responses, fixed AI transcript columns, and derived summary counts) — it does not include the raw AI-message-log or behavioral-event arrays. JSON returns every stored record exactly as captured, with nothing flattened or removed (including the full AI message log and behavioral events) — keep it as the authoritative archive.

If this server is running on Render with local-file storage (no Cloud Storage bucket configured), accumulated records live on that instance's local disk, which Render can clear on redeploy or restart. Download and back up your data before redeploying. See DATA_EXPORT.md for details and for the durable Google Cloud Storage setup.
